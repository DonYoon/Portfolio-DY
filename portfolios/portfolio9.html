Recipe Finder

Project Description: "Recipe Finder" is an interactive web application developed using React.js. This application allows users to search for meals, view details of individual meals, add meals to a favorite list, and remove meals from this list. All the user data is saved in the local storage, thus making it persist over sessions. The project is hosted on Replit, making it readily accessible without the need for deployment on other platforms. Clicking on the Replit link takes you straight to the live version of the application.

Features:

  • Search Component: This feature allows the users to input their desired meal in a search bar. The user can either enter a specific search term or click the "Surprise me!" button for a random meal suggestion.
  • Meals Component: This feature displays a list of meals that match the user's search term or a random meal if the user requested a surprise. Each meal is accompanied by an image and a like button. Clicking the like button adds the meal to the user's list of favorites. Clicking the image opens up a modal with more information about the meal.
  • Favorites Component: This feature displays a list of meals the user has liked. Each favorite meal is accompanied by an image and a remove button. Clicking the remove button removes the meal from the favorites list. Clicking the image opens up a modal with more details about the meal.
  • Modal Component: This feature displays detailed information about a selected meal. The details include the meal's image, name, and cooking instructions. A close button allows the user to close the modal.
  • App Context: The application leverages React's context API to manage state across components. This includes the current loading status, list of meals fetched from the API, search term, favorite meals, and selected meal for the modal.

Key Technologies:

  • React.js: A JavaScript library for building user interfaces, used for creating the various components in this application.
  • Context API: This feature of React.js is used to manage global state across multiple components without having to pass props down manually at every level.
  • Local Storage: This web storage object is used to store user data (favorite meals) across browser sessions.
  • Axios: A promise-based HTTP client for the browser and node.js, used to make API calls to TheMealDB.
